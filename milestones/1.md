## MatMul Optimization Benchmark

Shape: 1024 x 1024 x 1024  
Build: Release

| Implementation | Time |
|---|---:|
| naive_matmul | 8455 ms |
| fast_matmul | 298 ms |
| blocked_matmul_64 | 277 ms |
| parallel_matmul_8 | 65 ms |
| parallel_matmul_16 | 59 ms |
| threadpool_matmul_8 | 68 ms |

Key observations:

- Avoiding checked `at()` in the hot path gives the largest speedup.
- Row-wise parallelization scales well up to around 8 threads.
- ThreadPool has small scheduling overhead for one large matmul, but is more suitable for repeated runtime execution.
